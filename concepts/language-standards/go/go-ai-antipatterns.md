# Go AI-Generated Anti-Patterns

> Common patterns produced by AI that violate Go idioms. Load when reviewing AI-assisted code.

## Activation Triggers

Load this document when:
- Reviewing code generated by AI assistants
- Code seems over-engineered for its purpose
- Seeing factory patterns, excessive configuration, or unnecessary abstractions
- Code has verbose boilerplate that could be simpler

## Over-Engineering

```go
// AI SMELL: Factory pattern for single implementation
type ValidatorFactory interface {
    CreateValidator(validatorType string) Validator
}

type validatorFactory struct{}

func (f *validatorFactory) CreateValidator(validatorType string) Validator {
    switch validatorType {
    case "email":
        return &EmailValidator{}
    default:
        return &DefaultValidator{}
    }
}

// You only have one validator type! YAGNI.

// RIGHT: Direct construction
func NewEmailValidator() *EmailValidator {
    return &EmailValidator{}
}
```

## Unnecessary Channels

```go
// AI SMELL: Channel for synchronous operation
func ProcessItem(item Item) <-chan Result {
    resultChan := make(chan Result, 1)
    go func() {
        result := doWork(item)  // Synchronous work
        resultChan <- result
        close(resultChan)
    }()
    return resultChan
}

// Caller:
result := <-ProcessItem(item)  // Why channels?

// RIGHT: Simple function return
func ProcessItem(item Item) Result {
    return doWork(item)
}
```

## Wrong Abstraction Level

```go
// AI SMELL: Interface for implementation detail
type StringJoiner interface {
    Join(strings []string, separator string) string
}

type defaultStringJoiner struct{}

func (j *defaultStringJoiner) Join(strings []string, separator string) string {
    return strings.Join(strings, separator)
}

// RIGHT: Just use strings.Join directly
result := strings.Join(items, ",")
```

## Excessive Configuration

```go
// AI SMELL: Configuration for everything
type ClientConfig struct {
    BaseURL                string
    Timeout                time.Duration
    RetryCount             int
    RetryDelay             time.Duration
    MaxIdleConns           int
    IdleConnTimeout        time.Duration
    TLSHandshakeTimeout    time.Duration
    ResponseHeaderTimeout  time.Duration
    ExpectContinueTimeout  time.Duration
    // ... 15 more options
}

// RIGHT: Sensible defaults, configure only what varies
type Client struct {
    baseURL string
    http    *http.Client
}

type ClientOption func(*Client)

func WithTimeout(d time.Duration) ClientOption {
    return func(c *Client) {
        c.http.Timeout = d
    }
}

func NewClient(baseURL string, opts ...ClientOption) *Client {
    c := &Client{
        baseURL: baseURL,
        http:    &http.Client{Timeout: 30 * time.Second},
    }
    for _, opt := range opts {
        opt(c)
    }
    return c
}
```

## Reinventing Package Parsing

```go
// AI SMELL: Custom version parsing
func parseVersion(v string) (major, minor, patch int, err error) {
    parts := strings.Split(v, ".")
    if len(parts) != 3 {
        return 0, 0, 0, errors.New("invalid version")
    }
    major, err = strconv.Atoi(parts[0])
    // ... more parsing

// RIGHT: Use semver library
import "golang.org/x/mod/semver"

if semver.IsValid("v" + version) {
    major := semver.Major("v" + version)
}
```

## Verbose Error Handling

```go
// AI SMELL: Repetitive error handling
func Process() error {
    result1, err := step1()
    if err != nil {
        return fmt.Errorf("step1 failed: %w", err)
    }

    result2, err := step2(result1)
    if err != nil {
        return fmt.Errorf("step2 failed: %w", err)
    }

    result3, err := step3(result2)
    if err != nil {
        return fmt.Errorf("step3 failed: %w", err)
    }

    return step4(result3)
}

// RIGHT: Accept Go's error idiom - it's intentional
// But use shorter prefixes
func Process() error {
    result1, err := step1()
    if err != nil {
        return fmt.Errorf("step1: %w", err)  // Shorter
    }
    // ...
}
```

## Premature Caching

```go
// AI SMELL: Adding cache "for performance"
var cache = make(map[string]*Result)
var cacheMu sync.RWMutex

func GetResult(id string) (*Result, error) {
    cacheMu.RLock()
    if r, ok := cache[id]; ok {
        cacheMu.RUnlock()
        return r, nil
    }
    cacheMu.RUnlock()
    // ...
}

// Problems:
// - No TTL (stale data forever)
// - No max size (memory leak)
// - No metrics to know if cache helps
// - You don't even know if this is slow!

// RIGHT: Add caching when you measure a need
func GetResult(id string) (*Result, error) {
    return fetchFromSource(id)  // Start simple
}
```

## Unnecessary Generics

```go
// AI SMELL: Generics for non-generic code
type Repository[T any] interface {
    Get(id string) (T, error)
    Save(entity T) error
    Delete(id string) error
}

type UserRepository struct {
    // Only ever stores Users
}

// RIGHT: Concrete types unless you actually need generics
type UserRepository struct {
    db *sql.DB
}

func (r *UserRepository) Get(id string) (*User, error) {
    // ...
}
```

## Detection Checklist

Look for these AI-generated patterns:
- [ ] **[BLOCKER]** Factory patterns for single implementations
- [ ] **[BLOCKER]** Channels wrapping synchronous operations
- [ ] **[BLOCKER]** Caching without measurement (no TTL, no max size, no metrics)
- [ ] **[MAJOR]** Interfaces for stdlib operations
- [ ] **[MAJOR]** Configuration structs with 10+ fields
- [ ] **[MAJOR]** Custom parsing for well-known formats (use existing libraries)
- [ ] **[MAJOR]** Generics where concrete types work
- [ ] **[MINOR]** "Helper" functions that just wrap stdlib
- [ ] **[MINOR]** Error messages with "failed to" prefix
- [ ] **[MINOR]** Excessive documentation on simple functions
